# Data Model: Security Audit Fixes

**Branch**: `011-security-audit-fixes` | **Date**: 2026-03-20

## Schema Changes

### `rockport_video_jobs` table (db.py)

Only one schema change is required:

| Column | Current | After Fix | Reason |
|--------|---------|-----------|--------|
| `invocation_arn` | `TEXT UNIQUE NOT NULL` | `TEXT UNIQUE` (nullable) | CRIT-1: Slot must be reserved before Bedrock call, so ARN is unknown at insert time |
| `status` | `TEXT NOT NULL DEFAULT 'in_progress'` | `TEXT NOT NULL DEFAULT 'pending'` | New `pending` state for reserved-but-not-yet-submitted jobs |

### State Transitions

```
pending → in_progress    (Bedrock invocation succeeds, ARN is set)
pending → failed         (Bedrock invocation fails, slot released)
in_progress → completed  (Bedrock job finishes successfully)
in_progress → failed     (Bedrock job fails or times out)
```

### New DB Functions (db.py)

| Function | Purpose |
|----------|---------|
| `update_job_arn(job_id, invocation_arn)` | Set ARN and status='in_progress' after successful Bedrock call |
| `mark_job_failed(job_id, error_message)` | Set status='failed' and error message when Bedrock call fails |

## New Files

### `sidecar/requirements.lock`

Hashed lock file generated by `pip-compile --generate-hashes`. Contains all sidecar Python dependencies with SHA256 hashes for integrity verification.

### `terraform/cloudtrail.tf`

New Terraform resource file containing:
- S3 bucket for CloudTrail logs (with encryption, DenyNonSSL, 90-day lifecycle)
- CloudTrail trail (management events only)
- S3 bucket policy granting CloudTrail write access
